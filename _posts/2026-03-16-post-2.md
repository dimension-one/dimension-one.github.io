<div class="tldr">
  <strong>TL;DR.</strong>
  The previous post exhibited a situation where explaining the generalization gap using common complexity measures can be challenging:
  the same model and optimizer can interpolate every dataset yet generalize very differently depending on the labels.
  In this very same setup, however, the gap itself has a simple structure,
  so we know exactly what quantity a bound would need to capture.
  At interpolation it equals the population error, which decomposes into Bayes error plus a disagreement term with the Bayes classifier.
  In one dimension, that disagreement term can be computed exactly from the learned parameters.
</div>

In the previous post, 
we saw that the same model and optimizer can generalize very differently depending on the data.

But the data-generating process is usually unknown 
(which is precisely why we train a model to recover it in some sense), 
so we cannot directly inspect it to understand generalization.

After training, what we can actually inspect are the learned parameters.
If generalization depends on the data distribution, 
then that dependence must leave an observable footprint in those parameters.

This post asks: what is that footprint, exactly?
In a simple one-dimensional setting, 
it can be computed in closed form.

At interpolation (zero training error), 
we simply have 
{::nomarkdown}$$\text{generalization gap} = \text{population error} - \text{train error} = \text{population error}$$
{:/nomarkdown}
so the question becomes: *what is the population error equal to?*

---

## what actually determines population error?

We consider binary classification with inputs $X\in\mathbb{R}^d$ and labels {::nomarkdown}$Y\in\{-1,+1\}${:/nomarkdown}.

A classifier {::nomarkdown}$\hat y${:/nomarkdown} is a function {::nomarkdown}$\mathbb{R}^d\to\{-1,+1\}${:/nomarkdown}.
Its population error is
{::nomarkdown}
$$
\P(\hat y(X)\neq Y).
$$
{:/nomarkdown}

There exists an optimal classifier,
a **Bayes classifier** {::nomarkdown}$y^\star${:/nomarkdown}
which minimizes this quantity.
A Bayes classifier provides a natural reference point.

For any classifier {::nomarkdown}$\hat y${:/nomarkdown}, we can decompose its population error as
{::nomarkdown}
$$
\P(\hat y(X)\neq Y)
=
\P(y^\star(X)\neq Y) + \mathcal{E}(\hat y).
$$
{:/nomarkdown}
<!-- \underbrace{\P(y^\star(X)\neq Y)}_{\text{indep of }\hat y, \text{ Bayes error}}
+
\underbrace{\mathcal{E}(\hat y)}_{\text{depends on }\hat y, \text{ gap to Bayes}}. -->

The first term is the **Bayes error**, which is independent of the learned classifier {::nomarkdown}$\hat y${:/nomarkdown}. 
The second term depends on {::nomarkdown}$\hat y${:/nomarkdown} and measures the **excess error** relative to Bayes:
{::nomarkdown}
$$
\mathcal{E}(\hat y)
\coloneqq
\E\left[
\lvert 2\eta(X)-1\rvert \,
\cdot
\,
\mathbf{1}_{\{\hat y(X)\neq y^\star(X)\}}
\right],
\qquad
\eta(x)\coloneqq\P(Y=+1\mid X=x).
$$
{:/nomarkdown}

When training error is zero, 
the generalization gap is entirely captured by this excess error term, 
which measures the deviation from a Bayes classifier. 
Here, a convenient choice of Bayes classifier is 
{::nomarkdown}$$\text{Bayes classifier}(x)\coloneqq\operatorname{sign}(\eta(x)-1/2).$${:/nomarkdown}

<summary>
    <span>details <span class="hint">(click to expand)</span>: why $\operatorname{sign}(\eta(x)-1/2)$ is Bayes optimal and proof of the decomposition</span>
</summary>
<div class="details-body" markdown="1">

Fix {::nomarkdown}$x${:/nomarkdown}.
Once we condition on {::nomarkdown}$X=x${:/nomarkdown}, 
the prediction {::nomarkdown}$\hat y(x)${:/nomarkdown} is just a constant in {::nomarkdown}$\{-1,+1\}${:/nomarkdown}.
The only randomness left is in {::nomarkdown}$Y${:/nomarkdown}, so the conditional error is
{::nomarkdown}
$$
\P(\hat y(X)\neq Y\mid X=x)
=
\begin{cases}
\P(Y=-1\mid X=x), & \text{if }\hat y(x)=+1,\\\ 
\P(Y=+1\mid X=x), & \text{else if }\hat y(x)=-1.
\end{cases}
$$
{:/nomarkdown}

Since
{::nomarkdown}
$
\eta(x)=\P(Y=+1\mid X=x),
$
{:/nomarkdown}
we get the compact indicator form
{::nomarkdown}
$$
\P(\hat y(X)\neq Y\mid X=x) = (1-\eta(x)) \mathbf{1}_{\{\hat y(x)=+1\}} + \eta(x) \mathbf{1}_{\{\hat y(x)=-1\}}.
$$
{:/nomarkdown}

At a fixed {::nomarkdown}$x${:/nomarkdown}, 
an optimal prediction {::nomarkdown}$\hat y(x)${:/nomarkdown} is 
one that minimizes this conditional error.

- Predicting {::nomarkdown}$+1${:/nomarkdown} gives conditional error {::nomarkdown}$1-\eta(x)${:/nomarkdown}.
- Predicting {::nomarkdown}$-1${:/nomarkdown} gives conditional error {::nomarkdown}$\eta(x)${:/nomarkdown}.

Since {::nomarkdown}$1-\eta(x)\leq \eta(x)${:/nomarkdown} if and only if {::nomarkdown}$\eta(x)\geq 1/2${:/nomarkdown}, 
this shows that we can indeed choose a Bayes classifier of the form
{::nomarkdown}
$$
y^\star(x)=
\begin{cases}
+1, & \eta(x)\ge 1/2,\\
-1, & \eta(x)< 1/2,
\end{cases}
\qquad\text{i.e.}\qquad
y^\star(x)=\operatorname{sign}\big(\eta(x)-1/2\big)
\ \ (\text{ties arbitrary}).
$$
{:/nomarkdown}

We now prove the decomposition of the population error of any classifier {::nomarkdown}$\hat y${:/nomarkdown} into Bayes error + excess error. 
Fix {::nomarkdown}$x${:/nomarkdown} again.

- If {::nomarkdown}$\hat y(x)=y^\star(x)${:/nomarkdown}, then {::nomarkdown}$\hat y${:/nomarkdown} achieves the Bayes conditional error at {::nomarkdown}$x${:/nomarkdown}.
- If {::nomarkdown}$\hat y(x)\neq y^\star(x)${:/nomarkdown}, then {::nomarkdown}$\hat y(x)${:/nomarkdown} picks the *other* label, so its conditional error becomes
  {::nomarkdown}$\max\{\eta(x),1-\eta(x)\}${:/nomarkdown} instead of {::nomarkdown}$\min\{\eta(x),1-\eta(x)\}${:/nomarkdown}.
  The gap is therefore
  {::nomarkdown}
  $$
  \max\{\eta(x),1-\eta(x)\}-\min\{\eta(x),1-\eta(x)\}=\lvert 2\eta(x)-1\rvert.
  $$
  {:/nomarkdown}

So we have the pointwise identity
{::nomarkdown}
$$
\P(\hat y(X)\neq Y\mid X=x)
=
\P(y^\star(X)\neq Y\mid X=x)
+
\lvert 2\eta(x)-1\rvert \,\mathbf{1}_{\{\hat y(x)\neq y^\star(x)\}}.
$$
{:/nomarkdown}

Taking expectation over {::nomarkdown}$X${:/nomarkdown} gives, as claimed:
{::nomarkdown}
$$
\P(\hat y(X)\neq Y)
=
\P(y^\star(X)\neq Y)
+
\mathcal{E}(\hat y).
$$
{:/nomarkdown}
As a side note, even though we will not use it, 
since {::nomarkdown}$\lvert 2\eta(X)-1\rvert\le 1${:/nomarkdown}, 
we also obtain a cruder but more interpretable bound via the disagreement set:
{::nomarkdown}
$$
\mathcal{E}(\hat y)\le \P(\hat y(X)\neq y^\star(X)).
$$
{:/nomarkdown}
</div>
</details>

## why this is usually hard to compute

Ideally, there would be a ``simple'' recipe to compute the excess error of a learned classifier, given its parameters.
But in general, and especially for high-dimensional inputs $x$:

- the Bayes classifier {::nomarkdown}$\operatorname{sign}(\eta(x)-1/2)${:/nomarkdown} can be complicated, 
- the disagreement set {::nomarkdown}$\{x: \hat y(x) \neq y^\star(x)\}${:/nomarkdown} can have a complex geometry, 
- its measure under the data distribution can be intractable.

This is why people rely on indirect proxies:
margins, flatness, norms of the parameters, etc.

However, when the input is one-dimensional, exact computation is sometimes possible. 
The rest of this post is devoted to such an example.

---

## a setting where everything is explicit

Consider the 1d problem (the same as in the previous post):

- {::nomarkdown}$X\sim\mathrm{Unif}([-1,1])${:/nomarkdown}
- label {::nomarkdown}$y(x)\coloneqq \operatorname{sign}(x)${:/nomarkdown}
- with probability {::nomarkdown}$p${:/nomarkdown}, the label is replaced by an independent random sign {::nomarkdown}$\text{Unif}(\{-1,+1\})${:/nomarkdown}.

In this setting, an input {::nomarkdown}$x${:/nomarkdown} has probability {::nomarkdown}$1-p/2${:/nomarkdown} of having its label equal to {::nomarkdown}$\operatorname{sign}(x)${:/nomarkdown}, and probability {::nomarkdown}$p/2${:/nomarkdown} of having its label equal to the opposite sign. Therefore the function {::nomarkdown}$\eta(x)${:/nomarkdown} is

{::nomarkdown}
$$
\eta(x) = \begin{cases}
1-p/2, & \text{if } x>0,\\
p/2, & \text{else if } x<0,
\end{cases}
$$
{:/nomarkdown}

and the Bayes classifier {::nomarkdown}$y^\star(x)\coloneqq\operatorname{sign}(\eta(x)-1/2)${:/nomarkdown} 
is simply the sign rule {::nomarkdown}$y^\star(x)=\operatorname{sign}(x)${:/nomarkdown}.

<div class="details-body" markdown="1">

We know that a Bayes classifier is
{::nomarkdown}
$$
y^\star(x)=
\begin{cases}
+1, & \text{if }\eta(x)\ge 1/2,\\
-1, & \text{else if }\eta(x)< 1/2.
\end{cases}
$$
{:/nomarkdown}

From the expression of {::nomarkdown}$\eta(x)${:/nomarkdown} above, we have for every {::nomarkdown}$p\in[0,1]${:/nomarkdown}:
- if {::nomarkdown}$x>0${:/nomarkdown}, then {::nomarkdown}$\eta(x)=1-p/2${:/nomarkdown} and since this is at least 1/2, we have {::nomarkdown}$y^\star(x)=+1${:/nomarkdown}.
- otherwise {::nomarkdown}$x<0${:/nomarkdown}, in which case {::nomarkdown}$\eta(x)=p/2\le 1/2${:/nomarkdown}, hence {::nomarkdown}$y^\star(x)=-1${:/nomarkdown}.

Therefore,
{::nomarkdown}
$$
y^\star(x)=\operatorname{sign}(x)\quad \text{for all } p\in[0,1].
$$
{:/nomarkdown}

(at {::nomarkdown}$p=1${:/nomarkdown} we have {::nomarkdown}$\eta(x)=1/2${:/nomarkdown} everywhere, so every classifier is Bayes-optimal; the sign rule is still one of them.)
</div>
</details>

<div class="details-body" markdown="1">
- Corrupted labels are pure independent noise, so all rules perform equally on them: error is {::nomarkdown}$1/2${:/nomarkdown} for every classifier, including the sign rule.
- Clean labels are perfectly classified by the sign rule. 

This is why the sign rule is optimal in all cases.

In particular, the Bayes error is {::nomarkdown}$p/2${:/nomarkdown}, 
and the Bayes confidence is {::nomarkdown}$\lvert 2\eta(x)-1\rvert=1-p${:/nomarkdown} for all {::nomarkdown}$x${:/nomarkdown} 
(except at 0, which is irrelevant since {::nomarkdown}$X${:/nomarkdown} has no mass there).

Therefore, for this 1d problem, the population error of any classifier {::nomarkdown}$\hat y${:/nomarkdown} simplifies to
{::nomarkdown}
$$
\text{population error}(p)
=
p/2 +
(1-p)\,\P\big(\hat y(X)\neq \operatorname{sign}(X)\big).
$$
{:/nomarkdown}


In the previous post, we trained a one-hidden-layer ReLU network with a 2d output on this problem.
Let the network output be {::nomarkdown}$f(x)=(f_1(x),f_2(x))${:/nomarkdown}.
Define the logit gap
{::nomarkdown}
$$
\Delta(x)\coloneqq f_2(x)-f_1(x).
$$
{:/nomarkdown}

The learned classifier is {::nomarkdown}$\hat y(x)=\operatorname{sign}(\Delta(x))${:/nomarkdown}. 
Consider the Lebesgue measure of the disagreement set
{::nomarkdown}
$$
\mathrm{Leb\_wrong}
\coloneqq
\mathrm{Leb}\Big\{
x:
\operatorname{sign}(\Delta(x))\neq \operatorname{sign}(x)
\Big\}.
$$
{:/nomarkdown}

We obtain, in terms of the network's output:
{::nomarkdown}
$$
\boxed{
\text{population error}(p)
=
p/2
+
(1-p)\,\mathrm{Leb\_wrong}/2
}
$$
{:/nomarkdown}

## computing the disagreement set in 1d

For a one-hidden-layer ReLU network,
{::nomarkdown}$\Delta${:/nomarkdown} is piecewise affine: 
there is a partition of the input space into intervals such that
{::nomarkdown}
$$
\Delta(x)=mx+c
\quad\text{on each interval}.
$$
{:/nomarkdown}

The intervals are determined by the breakpoints where the ReLU units switch from inactive to active. 
For a hidden neuron implementing {::nomarkdown}$x\mapsto \max(0, w x + b)${:/nomarkdown}, 
its breakpoint is at
{::nomarkdown}
$$
x=-b/w
$$
{:/nomarkdown}


By collecting and sorting these breakpoints,
we obtain intervals on which {::nomarkdown}$\Delta${:/nomarkdown} is affine, 
and the associated slope {::nomarkdown}$m${:/nomarkdown} and intercept {::nomarkdown}$c${:/nomarkdown}. 

On each interval, 
we can compute the length of the region where the signs differ, 
and summing over intervals gives {::nomarkdown}$\mathrm{Leb\_wrong}${:/nomarkdown}.

In summary: 
parameters {::nomarkdown}$\to${:/nomarkdown} breakpoints {::nomarkdown}$\to${:/nomarkdown} partition of {::nomarkdown}$[-1,1]${:/nomarkdown} {::nomarkdown}$\to${:/nomarkdown} affine pieces of {::nomarkdown}$\Delta${:/nomarkdown} {::nomarkdown}$\to${:/nomarkdown} {::nomarkdown}$\mathrm{Leb\_wrong}${:/nomarkdown}.


Below is how this computation looks in code.

```python
net = nn.Sequential(
    nn.Linear(1, width, bias=True),
    nonlinearity,
    nn.Linear(width, 2, bias=False),
)
w = net[0].weight[:, 0]
b = net[0].bias
v = net[2].weight[1, :] - net[2].weight[0, :] # logit gap(x) = f_2(x) - f_1(x) = v^T ReLU(w x + b)

# breakpoints are the -b_j / w_j inside (-1, 1)
mask_w = (w.abs() > 1e-12) # if w_j ≈ 0, contributes a constant ReLU(b_j) to the output so no breakpoint (and it's a dead neuron if b_j < 0: could be removed)
bk = (-b[mask_w] / w[mask_w]).cpu().numpy()
bk = np.unique(bk[(bk > -1) & (bk < 1)])
bk = np.concatenate(([-1], bk, [1]))

Leb_wrong = 0.0

for i in range(len(bk) - 1):
    l, r = float(bk[i]), float(bk[i + 1]) # interval [left, right]

    mid = 0.5 * (l + r)
    act = ((w * mid + b) > 0).to(w.dtype)  # which ReLU neurons are active on this region
    # logit gap(x) = f_2(x) - f_1(x) = m * x + c on this region
    m = (v * act * w).sum().item()
    c = (v * act * b).sum().item()

    # Leb( sign(Delta(x)) != sign(x) )
    cuts = [(l, r)]
    if l < 0 < r:
        cuts = [(l, 0.0), (0.0, r)]
    for (ll, rr) in cuts:
        true_sign = (-1 if rr <= -1e-12 else (1 if ll >= 1e-12 else 0)) 
        if true_sign == 0:
            continue

        if abs(m) <= 1e-12:
            # constant Delta(x) = c on this interval
            if c * true_sign < -1e-12: # if c has opposite sign of true_sign, count the whole interval as wrong
                Leb_wrong += (rr - ll)
        else:
            x0 = -c / m # zero of mx+c, where it changes sign on that region
            if m > 0:
                pos_subinterval = max(0.0, rr - max(ll, x0)) # positive on x > x0
            else:
                pos_subinterval = max(0.0, min(rr, x0) - ll) # positive on x < x0
            Leb_wrong += pos_subinterval if true_sign == -1 else (rr - ll - pos_subinterval)
```

---

## what we observe

When we run this computation in the setup from the previous post:
- the training error is near zero for all label corruption levels
- the population error stays close to Bayes error {::nomarkdown}$p/2${:/nomarkdown}
- the disagreement length {::nomarkdown}$\mathrm{Leb\_wrong}${:/nomarkdown} can be computed exactly from the learned parameters using the code above, and when inserted into the analytical formula
{::nomarkdown}$$
\text{population error}(p)=p/2 + (1-p)\mathrm{Leb\_wrong}/2,$${:/nomarkdown}
it matches the estimated population error (up to Monte Carlo fluctuations from using a finite test set).

## takeaway

Generalization in a classification problem is about where 
the learned classifier deviates from the Bayes rule.

In high dimension,
this deviation is hard to characterize.

In one dimension, 
with a simple model, 
it can be computed exactly from the learned parameters.
